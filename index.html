Welcome to BlackLink Demo.

Use the links below to test out different deeplinks.


URI scheme

Either fails with error or does nothing when app is not installed.
We can't link directly to this, as we won't know if the app is installed.

Installed app: blacklane://splash
No app: blacklane2://splash


Chrome intent link

Wrapper around App link that can handle no installed app case.
Redirects on the phone to the play store or to a webpage.
Only works on phones with certain browsers !!, See chart.

Installed app: intent://splash#Intent;scheme=blacklane;package=com.blacklane.passenger.dev;end
No app with play store: intent://splash#Intent;scheme=blacklane;package=com.blacklane.passenger;end
No app with web fallback: intent://splash#Intent;scheme=blacklane;package=com.blacklane.passenger;S.browser_fallback_url=https%3A%2F%2Fwww.blacklane.com%2F;end


Javascript alternative to Chrome intent link

Runs a script on the webpage that redirects to app or play store.
This doesn't work on browsers that accept Chrome intent link.

Installed app: openApp("blacklane://splash")
No app with play store: openPlayStore("blacklane2://splash")
No app with web fallback: openWeb("blacklane2://splash")


Redirect Example

Redirects to Chrome Intent link if User-Agent contains "Android".
https://arcane-taiga-88765.herokuapp.com/


Dynamic link from Firebase

Firebase handles decision of redirecting to website, app or playstore.

Installed app: https://s5rt4.app.goo.gl/XktS
No app with play store: https://s5rt4.app.goo.gl/s2hO
No app with web fallback: https://s5rt4.app.goo.gl/IqL8


Dynamic link from Branch.io

Branch.io handles decision of redirecting to website, app or playstore.

Installed app: https://jqzw.test-app.link/mGtaKqbK7v
No app with play store: https://jqzw.app.link/6s9Zei6K7v
No app with web fallback: https://jqzw.app.link/v4MeeEZK7v


Url handling (App links)

App is configured to handle "https://www.uber.com/" links.
This will display a picker asking you if you want to open the app.
For Android 6.0+, we can skip the picker by verifying the site.
There is no server-side to this, and it is an Android-only feature.
